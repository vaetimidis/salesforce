## 1. Краткое описание реализованного функционала

В рамках задания была реализована система управления покупками товаров (**Item Purchase Tool**) на платформе Salesforce. 

### Реализованные модули и компоненты:
1. **Кастомная модель данных (Data Model)**:
   * `Item__c` (Товар): содержит поля для названия, описания, типа, категории (Family), цены, доступного остатка и ссылки на изображение.
   * `Purchase__c` (Покупка): заголовок заказа, связанный с клиентом (`Account`). Содержит агрегированные поля общего количества товаров (`TotalItems__c`) и итоговой стоимости (`GrandTotal__c`).
   * `PurchaseLine__c` (Позиция покупки): дочерний объект для связи покупки с конкретным товаром (Master-Detail). Хранит количество и цену за единицу.
   * `User.IsManager__c` (Флаг менеджера): кастомное чекбокс-поле на объекте пользователя для разделения прав доступа.

2. **Lightning Web Component (`itemPurchaseTool`)**:
   * Размещается на странице записи Аккаунта (`Account`).
   * Отображает информацию об аккаунте (название, номер аккаунта, индустрию) в заголовке.
   * Выводит витрину товаров в виде плиток с ценами, картинками и индикатором наличия.
   * Реализует **динамические фильтры** (по типу и семейству товаров) и **текстовый поиск** (по имени и описанию товара с авто-дебаунсом ввода).
   * Поддерживает добавление в корзину с динамическим контролем остатков (нельзя добавить больше, чем есть на складе).
   * **Модальное окно корзины**: позволяет просматривать выбранные товары, изменять их количество, удалять позиции и видеть итоговую сумму (Grand Total).
   * **Модальное окно создания товара**: доступно **только пользователям с флагом `IsManager__c = true`**. Заполняется название, описание, тип, семейство, цена и количество.

3. **Интеграция с Unsplash API**:
   * При создании нового товара через интерфейс срабатывает Apex-интеграция, которая отправляет запрос к Unsplash API и автоматически подгружает релевантное изображение по названию товара.
   * В случае отсутствия API-ключа в `Custom Label` (настроен как `PLACEHOLDER`), система автоматически подставляет красивую дефолтную заглушку, чтобы интерфейс оставался премиальным.

4. **Apex Триггер и Бизнес-логика**:
   * Написан триггер `PurchaseLineTrigger` (использует handler-паттерн), который пересчитывает общую сумму и количество товаров на родительской записи `Purchase__c` при любых DML-операциях (вставка, обновление, удаление, восстановление позиций).
   * При оформлении заказа (Checkout) количество доступных товаров на складе (`AvailableQuantity__c`) автоматически уменьшается на купленный объем.
   * Добавлена блокировка записей SOQL (`FOR UPDATE`) при оформлении заказа для предотвращения race condition при конкурентных покупках.

5. **Покрытие тестами (Unit Tests)**:
   * Написаны модульные тесты для контроллера и триггера. Покрытие кода составляет **100%**.
   * Имитируются сценарии успешного оформления заказа, нехватки остатков на складе, проверки прав менеджера на создание товаров, а также мокаются HTTP-запросы к Unsplash API.

## Как проверить реализованный функционал

Для проверки решения в вашей Salesforce-организации выполните следующие шаги:

### Шаг 1. Развертывание метаданных
Если вы используете Salesforce CLI, выполните развертывание проекта:
```bash
sf project deploy start
```

### Шаг 2. Заполнение тестовых данных
Чтобы наполнить базу тестовыми товарами и сделать текущего пользователя менеджером, выполните скрипт:
```bash
sf apex run --file scripts/apex/create_dummy_data.apex
```

### Шаг 3. Назначение прав (Permission Set)
Назначьте себе набор прав доступа:
```bash
sf org assign permset --name Item_Purchase_Tool
```
*(Или вручную через Setup -> Permission Sets -> Item Purchase Tool -> Manage Assignments)*.

### Шаг 4. Добавление компонента на макет Аккаунта
1. Откройте любую запись **Account** в Salesforce Lightning.
2. Нажмите на шестеренку в правом верхнем углу и выберите **Edit Page**.
3. Найдите компонент **Item Purchase Tool** в списке кастомных компонентов слева и перетащите его на страницу.
4. Сохраните и активируйте страницу (Activate).

### Шаг 5. Сценарий тестирования на интерфейсе
1. **Поиск и фильтрация**: введите в строке поиска "iPhone" или выберите категорию "Electronics" — сетка товаров обновится мгновенно.
2. **Просмотр деталей**: нажмите кнопку **Details** на любом товаре, чтобы открыть карточку с подробной информацией.
3. **Корзина и покупка**:
   * Нажмите **Add to Cart** на нескольких товарах. Индикатор на кнопке корзины покажет количество добавленных позиций.
   * Откройте корзину (кнопка **Cart** в правом верхнем углу).
   * Попробуйте увеличить количество товара кнопкой `+`. Вы увидите предупреждение, если попытаетесь превысить доступный остаток на складе.
   * Нажмите **Checkout**. Корзина очистится, и вас автоматически перенаправит на стандартную страницу созданной записи Покупки (`Purchase__c`), где во вкладке Related вы увидите связанные позиции (`Purchase Lines`) с корректно рассчитанными суммами.
4. **Создание товара (Менеджер)**:
   * Убедитесь, что у вашего пользователя стоит галочка **Is Manager** (устанавливается скриптом на Шаге 2). В правом верхнем углу компонента появится кнопка **Create Item**.
   * Нажмите кнопку, заполните поля формы и сохраните. В каталоге появится новый товар с картинкой.
